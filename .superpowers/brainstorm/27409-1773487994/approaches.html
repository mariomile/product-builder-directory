How should multi-select work inside the filter dialog?

All 3 keep the terminal aesthetic. Pick the interaction model that feels right.

A

Toggle Badges (current pattern, extended)

Badges work the same as today — but clicking multiple ones adds them all to the filter. Cyan = selected. Click again to remove.

// filter resources ──────────────────
--type
tool ✓ course ✓ article newsletter book video
--pillar
delivery ✓ design strategy
[clear filters] [apply (3)]

Pros

  • Familiar — evolves current UX
  • Scannable at a glance

Cons

  • No visual count per category
B

Checkbox Rows

Replace badges with checkbox rows. More conventional multi-select. Clearly communicates "pick many."

// filter resources ──────────────────
--type (2 selected)
[clear filters] [apply (2)]

Pros

  • Universally understood
  • Works well for long lists

Cons

  • Breaks terminal aesthetic
  • Heavier, form-like feel
C

Badge grid with live count header

Same toggle badges as Option A, but the group header shows a live count badge — e.g. --type [2] — making the multi-select intent obvious.

// filter resources ──────────────────
--type [2]
tool ✓ course ✓ article newsletter book
--pillar [0]
design delivery strategy
[clear filters] [apply (2)]

Pros

  • Badge pattern preserved
  • Count header makes multi-select obvious
  • Best balance of terminal feel + clarity

Cons

  • Slightly more complex to implement
